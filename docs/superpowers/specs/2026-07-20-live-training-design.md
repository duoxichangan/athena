# 实时训练与录像复盘并存设计

## 目标

为“篮眸”增加以电脑摄像头为输入的实时训练演示，同时完整保留现有上传录像、异步处理、AI 分析和训练沙盘能力。

## 范围与原则

- 不修改或删除既有录像接口：`/upload`、`/status/{task_id}`、`/data/{task_id}`、`/analyze/{task_id}/{track_id}` 及其处理流程。
- 实时功能使用独立前端组件和独立后端接口，与录像任务状态隔离。
- 初版面向单台电脑摄像头、单浏览器演示；视频仅在当前浏览器预览，后端只接收用于分析的抽样帧。

## 用户界面

首页提供两个并列入口：

1. **实时训练**：用户授权电脑摄像头后看到实时画面、训练状态和右上角半场小地图；可暂停、结束、展开沙盘。
2. **录像复盘**：保留现有上传和全部结果页面，供赛后完整分析。

实时主画面右上角放置半场小地图。地图显示已追踪球员的教练友好名称、当前位置和最近 3–5 秒轨迹；点击小地图可打开现有沙盘样式的放大视图。

## 数据流

```text
电脑摄像头 → 浏览器 MediaDevices → 本地视频预览
                              └→ 定时抓帧（约 5–8 FPS）
                                   → POST /live/frame
                                   → YOLO 姿态与追踪
                                   → JSON 球员位置/骨架
                                   → 叠加层与半场小地图
```

初版使用浏览器画面中的归一化坐标绘制小地图，并明确标为“相对位置”。后续可增加四点手动场地标定，将画面坐标透视映射到真实半场坐标。

## 后端边界

- 新建实时会话状态与 `POST /live/start`、`POST /live/frame`、`POST /live/stop`。
- 每个会话维护独立跟踪器，避免与录像任务共享状态。
- 视频帧不落盘；结束会话立即清理状态。
- 若摄像头、解码或模型推理失败，接口返回清晰错误信息，不影响录像分析服务。

## 前端边界

- 新增 `LiveTraining`、`CourtMiniMap` 等组件，不修改已有上传分析组件的内部流程。
- 通过浏览器 `getUserMedia` 获取摄像头权限；拒绝权限时给出恢复指引。
- 网络或接口失败时继续显示本地画面，并提示实时标注暂不可用。

## 验收与测试

- 未开启实时训练时，上传录像分析流程与当前行为一致。
- 摄像头授权后，可看到实时预览；后端返回的球员位置会更新小地图。
- 结束实时训练会关闭媒体轨道、停止抓帧并清理实时会话。
- 为坐标转换和小地图渲染增加单元测试；为实时接口增加最小接口测试。
